## Цель работы и описание задачи

Цель работы — разработать на языке Go эмулятор протокола TokenRing, в котором узлы сети образуют логическое кольцо и передают друг другу сообщения в виде токенов.

Аргументом командной строки задается количество узлов в кольце. Каждый узел представляет собой отдельную горутину и взаимодействует с соседними узлами через каналы. Узел с номером i получает данные от узла i-1 и передает их узлу i+1.

Каждый токен содержит:
- текст сообщения;
- хэш SHA3-256 идентификатора получателя;
- счетчик времени жизни TTL;
- идентификатор отправителя;
- уникальный номер сообщения;
- путь прохождения сообщения по узлам.

Сообщение передается по кольцу до тех пор, пока не будет доставлено адресату или пока не истечет TTL. Первое сообщение отправляет основной поток узлу 1. После доставки сообщения соответствующий узел генерирует новое сообщение и отправляет его дальше. Во время работы программы в консоль выводится подробный лог пересылки сообщений.



## Реализованный функционал

### Структура `Token`

```go
type Token struct {
    ID       int
    Data     string
    DstID    int
    DstHash  byte[1]
    TTL      int
    OriginID int
    Path     []int
    Created  time.Time
}
```

Структура `Token` хранит все данные, необходимые для передачи сообщения по кольцу: содержимое, получателя, TTL, отправителя, путь и время создания.

### Структура `Node`

```go
type Node struct {
    ID        int
    In        chan Token
    Out       chan Token
    Processed atomic.Int64
}
```

Структура `Node` описывает узел кольца.  
Она хранит:
- номер узла;
- канал входящих сообщений;
- канал для передачи следующему узлу;
- счетчик обработанных сообщений.

### Глобальные переменные

```go
var (
    startTime   time.Time
    msgCounter  atomic.Int64
    totalCount  atomic.Int64
    uniqueCount atomic.Int64
    printMu     sync.Mutex
    rngMu       sync.Mutex
    rng         = rand.New(rand.NewSource(time.Now().UnixNano()))

```

Глобальные переменные используются для:
- синхронизации вывода в консоль;
- генерации уникальных идентификаторов сообщений;
- подсчета обработанных и созданных сообщений;
- хранения времени начала симуляции;
- генерации случайных чисел.

### Основные функции

- `hashID(id int) [32]byte` — вычисляет SHA3-256 хэш идентификатора узла.
- `ts() string` — возвращает время с начала симуляции в миллисекундах.
- `logf(format string, args ...any)` — выводит лог с синхронизацией.
- `randInt(n int) int` — возвращает случайное число.
- `randomData() string` — выбирает случайный текст сообщения.
- `makeToken(originID, dstID, ttl int, data string, path []int) Token` — создает новый токен.
- `nodeName(id int) string` — возвращает строковое имя узла.
- `func (n *Node) run(nodes int, stop <-chan struct{}, all []*Node, wg *sync.WaitGroup)` — основная логика работы узла.
- `main()` — создает кольцо, запускает узлы, отправляет первое сообщение и выводит статистику.


## Алгоритм работы узла

1. Узел ожидает сообщение в канале `In`.
2. После получения сообщение добавляет текущий узел в `Path`.
3. Выводится лог получения сообщения.
4. Если текущий узел является получателем, сообщение считается доставленным.
5. После доставки создается новое сообщение с новым получателем и случайным TTL.
6. Если TTL стал меньше или равен 1, сообщение удаляется.
7. Если сообщение не доставлено и TTL еще не истек, оно пересылается следующему узлу.
8. Работа узла завершается после сигнала остановки.
