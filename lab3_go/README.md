# Token Ring Simulation

## Описание задачи

В данной работе была реализована имитация работы кольцевой сети Token Ring на языке Go.  
Программа моделирует передачу сообщений между узлами, расположенными в логическом кольце.  
Каждое сообщение содержит данные, адрес получателя, TTL, идентификатор и путь прохождения по узлам.

Основная цель работы — показать, как сообщение передается по кольцу до своего адресата, а также как узлы могут создавать новые сообщения после успешной доставки.

## Как работает программа

Программа запускает заданное число узлов, по умолчанию 5.  
Каждый узел работает в отдельной goroutine и связан с соседними узлами через каналы.  
Сообщение передается от одного узла к следующему по кругу, пока не будет доставлено получателю или пока не истечет TTL.

Работа программы происходит в несколько этапов:

1. Создается кольцо узлов.
2. Главное приложение отправляет первое сообщение узлу 1.
3. Каждый узел принимает сообщение, добавляет себя в путь и проверяет, является ли он получателем.
4. Если сообщение дошло до адресата, оно считается доставленным.
5. После доставки узел создает новое сообщение и отправляет его дальше.
6. Если TTL становится равным нулю, сообщение удаляется.
7. Через заданное время симуляция завершается, и выводится статистика.

Для корректного завершения программы используется остановка симуляции через отдельный канал, а узлы завершают работу после получения сигнала остановки. Такой подход соответствует обычной практике graceful shutdown в Go [web:168][web:193].
